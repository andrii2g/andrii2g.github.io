# Codex AI Rules

These rules MUST be followed by any AI agent generating repositories or pages.

---

## MUST

- follow naming convention
- generate 3–7 valid tags
- assign at least 1 category
- create repository page
- update mkdocs.yml navigation
- update category page

---

## MUST NOT

- invent new naming styles
- create inconsistent tags
- skip navigation updates
- create duplicate categories

---

## Naming enforcement

Preferred format:

[prefix]-[domain]-[purpose]

Allowed prefixes:
- tool
- script
- dotnet
- cpp
- python
- web
- api
- infra
- devops
- iot
- embedded
- print
- home
- lab
- data
- docs

---

## Tag enforcement

Each generated repository entry should include:

- at least 1 tech tag
- at least 1 domain tag
- at least 1 type tag

Recommended tag count:
- 3 to 6 tags

---

## Output format

AI must generate:

1. repository name
2. repository description
3. tags
4. category
5. markdown page
6. navigation update

---

## Example

Repository:
tool-telegram-sticker-viewer

Tags:
- dotnet
- telegram
- tool

Category:
Telegram
