# Tags Guidelines

## Rules

- lowercase only
- 3–7 tags per repository
- predefined vocabulary only
- no duplicates or synonyms

---

## Required composition

Each repo MUST include:

- 1 tech tag
- 1 domain tag
- 1 type tag

---

## Tech tags

dotnet, cpp, python, blazor, minimal-api, mysql, redis, kafka, docker, k8s

---

## Domain tags

telegram, iot, 3d-printing, aws, vr-forces, mak-rti, home-automation

---

## Type tags

tool, service, library, script, experiment, prototype

---

## Example

```yaml
tags:
  - dotnet
  - telegram
  - tool
```

---

## Forbidden
- uppercase
- random tags
- synonyms (csharp + dotnet)
-more than 10 tags
