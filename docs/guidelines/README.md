# Repository Catalog Guidelines

This section defines all rules and conventions for managing repositories and catalog entries.

These guidelines MUST be followed for:
- repository creation
- naming
- tagging
- categorization
- documentation
- navigation updates

## Structure

- Naming → naming.md
- Tags → tags.md
- Categories → categories.md
- Repository Pages → repository-page.md
- Workflow → workflow.md
- Codex Rules → codex-rules.md

## Core Principle

Consistency > Clarity > Brevity > Creativity

## Purpose

This system is designed to scale across:

10 → 100 → 500+ repositories

Without these rules, the catalog becomes inconsistent and hard to maintain.
