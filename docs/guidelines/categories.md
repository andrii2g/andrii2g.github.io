# Categories Guidelines

## Rules

- categories must remain broad and stable
- do not create a new category unless the repository clearly does not fit existing categories
- categories are high-level grouping only
- one repository may belong to multiple categories
- each repository must belong to at least 1 category

## Category Usage

Each repository must:

- belong to at least 1 category
- be listed manually in category page
- use the most specific existing categories available
- avoid category spam

Recommended range:

- 1 to 3 categories per repository

## Do Not

- create duplicate categories with similar meaning
- create overly narrow categories for one repo only
- use categories instead of tags for detailed attributes

Bad:
- Telegram Bots
- Telegram APIs
- Telegram Sticker Tools

Better:
- Telegram

Use tags for the details instead.
